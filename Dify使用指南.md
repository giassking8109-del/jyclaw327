# 研发效能分析 - Dify 工作流使用指南

## 📋 概述

本工作流用于自动分析 Excel 需求数据，生成个人和团队的研发效能分析报告。

## 🎯 分析维度

### 个人效能分析
- 负责需求数量
- 完成率
- 工时效率（实际/预估比）
- 风险提示
- 改进建议

### 团队效能分析
- 团队需求总量
- 完成率排名
- 平均工时
- 风险等级分布
- 团队对比

### 综合报告
- 执行摘要
- 关键指标仪表板
- 个人效能排名
- 团队效能排名
- 风险预警
- 改进建议

## 🚀 导入步骤

### 方式一：手动创建（推荐）

1. **登录 Dify**
   打开你的 Dify 地址，登录账号

2. **创建空白应用**
   - 点击「创建应用」
   - 选择「导入DSL文件」或手动创建

3. **构建工作流**
   按以下顺序添加节点：

   ```
   开始 → Excel读取 → 数据处理 → 个人分析 → 团队分析 → 综合报告 → 结束
   ```

4. **配置变量**
   在「开始」节点添加：
   - `excel_file` (文件类型)
   - `analysis_type` (选择: 个人分析/团队分析/综合报告)

5. **发布并测试**
   上传 Excel 文件，运行工作流

### 方式二：使用知识库

1. **上传 Excel 到知识库**
   - 创建知识库
   - 上传「团队需求明细.xlsx」
   - 等待向量化完成

2. **创建对话应用**
   - 使用知识库检索
   - 使用下方提示词模板

## 📝 提示词模板

### 数据提取提示词
```
你是一个研发效能数据分析师。请从以下Excel数据中提取关键指标：

{Excel内容}

请提取并计算：
1. 各状态需求数量（已完成/进行中/待开发/规划中）
2. 各优先级分布（P0/P1/P2/P3）
3. 各团队需求数量
4. 工时统计（总预估工时、总实际工时、平均工时）
5. 完成率
6. 超时风险需求

以JSON格式输出。
```

### 个人分析提示词
```
基于以下数据，为每位负责人生成个人研发效能分析：

{结构化数据}

分析维度：
1. 负责需求数量

3. 工时效率（实际/预估）
4. 风险提示
5. 改进建议

```

### 团队分析提示词
```

{结构化数据}

分析维度：
1. 团队需求总量
```

## 📊 Excel 数据格式要求

| 列名 | 类型 | 说明 |
|------|------|------|
| 版本名称 | 文本 | 如 AB-CRPS1.0.1_101 |
| 需求描述 | 文本 | 功能描述 |
| 经理 | 文本 | 项目经理 |
| 负责人 | 文本 | 开发者 |
| 团队 | 文本 | 团队名称 |
| 业务域 | 文本 | 业务领域 |
| 系统 | 文本 | 系统代号 |
| 优先级 | 文本 | P0/P1/P2/P3 |
| 风险等级 | 文本 | 高/中/低 |
| 需求类型 | 文本 | 常规版/优化版/重构版 |
| 状态 | 文本 | 已完成/进行中/待开发/规划中 |
| 预估工时(h) | 数字 | 预估小时数 |
| 实际工时(h) | 数字 | 实际小时数 |
| 计划投产 | 日期 | 计划上线日期 |
| 实际投产 | 日期 | 实际上线日期 |
| 备注 | 文本 | 特殊情况 |

## ⚠️ 注意事项

1. Excel 第一行是表头，从第二行开始是数据
2. 确保日期格式统一（推荐 YYYY-MM-DD）
3. 数字列不要包含单位（如"100h"应写为"100"）
4. 状态字段必须使用：已完成/进行中/待开发/规划中

## 🎨 输出示例

### 个人效能报告
```

### 1. 概况
- 负责需求：12个
- 已完成：8个
- 进行中：2个
- 待开发：2个

### 2. 效能指标
- 完成率：66.7%
- 工时效率：92%（实际/预估）

### 4. 改进建议
✅ 建议加强需求前期评估，减少后期变更
```

```
- 低风险：8个
```
